# i18n（多言語対応）設計ドキュメント

**作成日:** 2026-04-14

---

## 概要

RMTsite に英語・日本語・中国語（簡体字）の3言語対応を追加する。デフォルト言語は英語。URL にロケールを含む形式（`/en/`, `/ja/`, `/zh/`）で実装し、ヘッダーに言語切り替えボタンを追加する。

---

## 技術スタック追加

- **next-intl** — Next.js 14 App Router 向け i18n ライブラリ

---

## URL 設計

| URL | 言語 |
|---|---|
| `/` | `/en` にリダイレクト（デフォルト） |
| `/en` | 英語（デフォルト） |
| `/ja` | 日本語 |
| `/zh` | 中国語（簡体字） |
| `/en/category` | 英語のカテゴリ一覧 |
| `/ja/category` | 日本語のカテゴリ一覧 |

ブラウザのAccept-Languageヘッダーは無視し、常に `/en` にリダイレクトする。

---

## ディレクトリ構造

```
app/
├── [locale]/              # ロケールルートグループ（en / ja / zh）
│   ├── layout.tsx         # 現 app/layout.tsx を移動・更新
│   ├── page.tsx           # 現 app/page.tsx を移動・更新
│   └── category/
│       └── page.tsx       # 現 app/category/page.tsx を移動・更新
└── globals.css            # そのまま

middleware.ts              # ルート直下に新設（ロケール検出・リダイレクト）

i18n/
├── routing.ts             # ロケール定義（locales, defaultLocale）
└── request.ts             # next-intl サーバー設定

messages/
├── en.json                # 英語テキスト（デフォルト）
├── ja.json                # 日本語テキスト
└── zh.json                # 中国語（簡体字）テキスト

components/
└── LanguageSwitcher.tsx   # 言語切り替えボタン（新規追加）
```

## コンポーネント設計

### LanguageSwitcher.tsx（新規）

```
"use client"
- usePathname() で現在のパスを取得
- useLocale() で現在のロケールを取得
- EN / JA / ZH の3ボタン
- 現在のロケールは bg-white text-blue-600（ハイライト）
- 他は text-white/70 hover:text-white
- クリックで pathName のロケール部分を置換して router.push()
```

### Header.tsx（更新）

- `useTranslations('header')` でテキストを取得（Server Component のまま）
- LanguageSwitcher を右端に追加
- 翻訳キーで全テキストを置換

### 更新が必要なコンポーネント

| コンポーネント | 変更内容 |
|---|---|
| `Header.tsx` | 翻訳適用 + LanguageSwitcher 追加 |
| `Footer.tsx` | 翻訳適用 |
| `Carousel.tsx` | バナーテキストを翻訳から取得（use client のまま） |
| `GameRankingTable.tsx` | `title` prop を呼び出し元から渡すので変更不要 |
| `Sidebar.tsx` | 翻訳適用 |
| `app/[locale]/page.tsx` | 翻訳適用（セクション見出し） |
| `app/[locale]/category/page.tsx` | 翻訳適用（カテゴリラベル・ページタイトル） |

## next-intl 設定ファイル

### i18n/routing.ts
```typescript
import { defineRouting } from 'next-intl/routing'

export const routing = defineRouting({
  locales: ['en', 'ja', 'zh'],
  defaultLocale: 'en',
  localeDetection: false  // ブラウザ言語を無視してen固定
})
```

### middleware.ts
```typescript
import createMiddleware from 'next-intl/middleware'
import { routing } from './i18n/routing'

export default createMiddleware(routing)

export const config = {
  matcher: ['/((?!_next|_vercel|.*\\..*).*)']
}
```

## テスト方針

- 既存の15テストはコンポーネントへの翻訳適用で影響を受ける可能性がある
- 翻訳後は各テストをロケールプロバイダー（`NextIntlClientProvider`）でラップして修正する
- `en` ロケールの翻訳文字列でテストを更新する
